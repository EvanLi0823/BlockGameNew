# 质询问题库

## 问题分类框架

### A类: 实现方案 (How & Why)
探索设计的实现方式和选择理由

### B类: 影响分析 (Impact)
评估对现有系统的影响

### C类: 风险识别 (Risk)
识别潜在问题和Bug风险

### D类: 边界验证 (Boundary)
检查边界情况处理

### E类: 可维护性 (Maintainability)
评估代码质量和可维护性

---

## A类: 实现方案问题

### A1: 技术选型
- "为什么选择[技术/模式]?有评估过其他方案吗?"
- "这个技术栈是否符合项目规范?"
- "团队对这个技术是否熟悉?"

### A2: 实现细节
- "[功能]具体如何实现?使用哪些Unity组件?"
- "数据结构如何设计?为什么?"
- "生命周期如何管理?Awake/Start/Update如何使用?"

### A3: API设计
- "对外暴露哪些接口?为什么?"
- "接口签名是否合理?参数是否足够/过多?"
- "如何处理错误?返回null还是抛异常?"

### A4: 数据流
- "数据从哪里来?PlayerPrefs/SaveFile/Network/配置?"
- "数据如何处理?验证/转换/计算?"
- "数据如何通知UI更新?事件/轮询/绑定?"

---

## B类: 影响分析问题

### B1: 向后兼容性
- "这个改动会影响现有功能吗?哪些?"
- "公共API是否改变?现有调用者如何迁移?"
- "是否需要提供兼容层?"

### B2: 依赖变化
- "这个功能依赖哪些现有系统?"
- "是否需要修改现有Manager的接口?"
- "是否有循环依赖风险?"

### B3: 性能影响
- "是否引入性能瓶颈?在哪里?"
- "是否频繁创建临时对象(GC压力)?"
- "Update调用是否可以优化?"

### B4: 数据迁移
- "旧数据如何处理?是否需要迁移?"
- "数据格式变化时如何兼容?"
- "是否需要版本号管理?"

---

## C类: 风险识别问题

### C1: 空引用风险
- "哪些地方可能出现NullReferenceException?"
- "是否所有引用类型都检查了null?"
- "是否使用了?.运算符或提供默认值?"

### C2: 时序问题
- "初始化顺序是否有依赖?如何确保?"
- "场景切换时如何处理?是否清理资源?"
- "异步操作中对象被销毁怎么办?"

### C3: 并发问题
- "是否存在协程/异步操作的竞争条件?"
- "多个异步操作如何协调?"
- "是否需要并发控制?"

### C4: 状态一致性
- "数据在哪里修改?如何保证一致性?"
- "是否有唯一数据源(Single Source of Truth)?"
- "UI状态与数据状态如何同步?"

---

## D类: 边界验证问题

### D1: 数据边界
- "当[参数]为null时,会发生什么?"
- "空列表/数组时如何处理?"
- "极值(int.MaxValue/0/负数)时如何处理?"

### D2: 时序边界
- "组件未初始化时调用会怎样?"
- "场景加载过程中调用会怎样?"
- "快速重复调用(连击)会怎样?"

### D3: 状态边界
- "GameObject被销毁后引用会怎样?"
- "GameObject禁用时逻辑是否正确?"
- "游戏暂停(timeScale=0)时是否有问题?"

### D4: 平台边界
- "不同分辨率/屏幕比例如何适配?"
- "低性能设备如何处理?"
- "不同Unity版本是否兼容?"

---

## E类: 可维护性问题

### E1: 代码可读性
- "命名是否清晰?符合规范吗?"
- "是否需要添加注释说明复杂逻辑?"
- "代码结构是否清晰?"

### E2: 可测试性
- "如何验证这个功能正确性?"
- "是否方便单元测试?"
- "依赖能否Mock?"

### E3: 可扩展性
- "未来新增需求时,需要修改现有代码吗?"
- "是否符合开闭原则(对扩展开放,对修改封闭)?"
- "哪些部分应该配置化?"

### E4: 可调试性
- "是否有足够的Debug.Log?"
- "错误信息是否清晰?"
- "是否提供Inspector调试选项?"

## 对话式提问技巧

### 1. 分层提问
```
第一层(宏观): "整体架构是怎样的?"
第二层(中观): "XX模块如何实现?"
第三层(微观): "这个方法为什么这样写?"
```

### 2. 引导式提问
```
❌ "这里有问题"
✅ "在XX场景下,当YY为null时,这段代码会如何执行?
   我注意到可能触发NullReferenceException。
   有考虑过添加null检查吗?"
```

### 3. 对比式提问
```
"关于XX的实现,我看到有两种方案:
 方案1: [描述] - 优点:XX 缺点:YY
 方案2: [描述] - 优点:AA 缺点:BB
 你选择了方案X,考虑是什么?有评估过另一个方案吗?"
```

### 4. 场景化提问
```
"假设用户在游戏过程中突然退出,
 然后重新进入,这个功能会如何表现?
 数据是否正确恢复?"
```

## 提问模板

### 实现方案模板
```
Q: "[功能]如何实现?"
期望了解: 技术选型、实现步骤、数据结构
风险提示: 实现不明确可能导致返工

Q: "为什么选择[方案]?"
期望了解: 选择理由、方案对比、权衡考虑
风险提示: 未充分评估可能选择次优方案
```

### 影响分析模板
```
Q: "这个改动会影响[XX功能]吗?"
期望了解: 影响范围、兼容性策略
风险提示: 影响评估不足可能破坏现有功能

Q: "是否需要同步修改[YY代码]?"
期望了解: 关联修改点、遗漏检查
风险提示: 遗漏关联修改可能导致编译错误
```

### 风险识别模板
```
Q: "当[XX条件]时,是否可能出现[YY问题]?"
期望了解: 异常处理、防御措施
风险提示: 未处理异常可能导致Crash

Q: "如何避免[ZZ风险]?"
期望了解: 预防措施、缓解策略
风险提示: 风险未缓解可能影响稳定性
```

### 边界验证模板
```
Q: "在[边界场景]下,如何处理?"
期望了解: 边界逻辑、容错机制
风险提示: 边界未处理可能导致异常

举例:
- "当列表为空时,如何处理?"
- "当配置缺失时,如何处理?"
- "当网络超时时,如何处理?"
```
